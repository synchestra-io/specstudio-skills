# Ideation Frameworks

Adapted from `addyosmani/agent-skills/skills/idea-refine/frameworks.md`. Use these frameworks selectively. Pick the lens that fits the idea — don't mechanically run every framework.

## SCAMPER

Transform an existing idea by applying seven operations:

- **Substitute** — swap a component, material, technology, audience, or business model.
- **Combine** — merge with another product, service, or idea.
- **Adapt** — borrow ideas from other industries, domains, time periods, or nature.
- **Modify (Magnify/Minimize)** — make it 10x bigger or smaller; exaggerate or strip one feature.
- **Put to other uses** — different users, different problems, different contexts.
- **Eliminate** — remove a feature entirely; zero-config; half the steps.
- **Reverse/Rearrange** — opposite order; user does the work instead of the system; reversed value chain.

**Best for:** improving or reimagining existing products/features. Less useful for greenfield ideas.

## How Might We (HMW)

Reframe problems as opportunities: "How might we [desired outcome] for [specific user] without [key constraint]?"

**Good HMW qualities:**
- Narrow enough to be actionable.
- Broad enough to allow creative solutions.
- Contains a tension or constraint that forces creativity.

**Bad HMW qualities:**
- Too broad ("…make users happy").
- Too narrow ("…add a button to settings").
- Solution-embedded ("…build a chatbot for support").

**Best for:** reframing stuck thinking; pulling people back from solutions to the problem.

## First Principles

Break the idea down to fundamental truths, then rebuild:

1. What do we know is true (not assumed)?
2. What are we assuming?
3. Which assumptions can we challenge?
4. Rebuild from the truths.

**Best for:** breaking out of incremental thinking.

## Jobs to Be Done (JTBD)

Focus on what the user is *hiring* the product to do:

- **Functional job** — what task are they completing?
- **Emotional job** — how do they want to feel?
- **Social job** — how do they want to be perceived?

Format: "When I [situation], I want to [motivation], so I can [outcome]."

**Key insight:** the competing product isn't always in the same category (Netflix competes with sleep).

**Best for:** understanding the real problem.

## Constraint-Based Ideation

Impose constraints to force creative solutions:

- **Time** — 1 day?
- **Feature** — only one feature?
- **Tech** — can't use the obvious technology?
- **Cost** — free forever?
- **Audience** — user has never used a computer?
- **Scale** — 1 billion users? Just 10?

**Best for:** cutting through complexity.

## Pre-mortem

Imagine the idea has already failed. Work backwards:

1. 12 months from now, the project flopped. What went wrong?
2. List every plausible failure — technical, market, team, timing.
3. For each: preventable? Signal the idea needs to change?
4. Which failures would kill the project?

**Best for:** Phase 2 evaluation — stress-testing ideas that feel good but haven't been pressure-tested.

## Analogous Inspiration

Look at how other domains solved similar problems:

- What industry has already solved a version of this?
- What would this look like if [specific company] built it?
- What natural system works this way?
- What historical precedent exists?

Find *structural* similarities, not surface-level ones. "Uber for X" is surface. "A two-sided marketplace solving a trust problem between strangers" is structural.

**Best for:** Phase 1 expansion — generating genuinely different variations.
